Everyday Git

Welcome to "Everyday Git," your go-to guide for mastering Git and version control!

1. What is Git?

Git is a distributed version control system (VCS) that helps developers track changes in their codebase. A version control system allows you to save snapshots of your project at different stages, making it easy to revert to a previous state if something goes wrong. Other VCS tools include Mercurial and Subversion, but Git is the most widely used.

2. Installing Git

To check if Git is installed on your machine, open your terminal and run:

git --version

If Git is not installed, download it from git-scm.com and follow the installation instructions for your operating system.

3. Configuring Git

After installing Git, configure your username and email address using the following commands:

Everyday Git

Welcome to "Everyday Git," your go-to guide for mastering Git and version control!

1. What is Git?

Git is a distributed version control system (VCS) that helps developers track changes in their codebase. A version control system allows you to save snapshots of your project at different stages, making it easy to revert to a previous state if something goes wrong. Other VCS tools include Mercurial and Subversion, but Git is the most widely used.

2. Installing Git

To check if Git is installed on your machine, open your terminal and run:

git --version

If Git is not installed, download it from git-scm.com and follow the installation instructions for your operating system.

3. Configuring Git

After installing Git, configure your username and email address using the following commands:

git config --global user.name "Your Name"
git config --global user.email "your.email@example.com"

4. Initializing a Git Repository

A Git repository is a folder where Git tracks changes to your files. To initialize a repository, navigate to your project folder and run:

git init

5. Adding Files to the Staging Area

The staging area is where you prepare files for a commit. To add files, use:

git add . # adds all files

Tracked files are files Git is aware of, while untracked files are not yet tracked by Git.

6. Committing Changes

To save your changes, commit them with a meaningful message:

git commit -m "Your commit message"

To view past commits, use:

git log

7. Branches

Branches allow you to work on different versions of your project simultaneously. The default branch is usually called main or master.

8. Creating a Branch

To create a new branch, use:

git branch branch-name

9. Viewing All Branches

To view all branches in your repository, run:

git branch

The active branch is marked with an asterisk (*).

10. Switching Active Branches

To switch to another branch, use:

git checkout branch-name

11. Merging a Branch

To merge a branch into the active branch, use:

git merge branch-name

12. Deleting a Branch

To delete a branch, run:

git branch -d branch-name

13. Next Steps

Now that you know the basics of Git, you can explore advanced topics like rebasing, resolving merge conflicts, and using GitHub for collaboration. Happy coding!
